# The Self-Managed Building

The goal is simple: every operational loop in the building runs without a human initiating it.

- Maintenance request comes in — triaged, vendor dispatched, resident updated, cost logged
- Lease expires — renewal outreach sent, concession calculated, renewal or re-listing triggered
- Occupancy drops — pricing adjusted, marketing spend shifted, leasing cadence increased
- Resident goes quiet for 60 days — proactive check-in sent, event invitation personalized
- NOI misses budget — variance flagged, root cause identified, corrective action proposed

Each of those is a loop. A self-managed building is what you get when all of them run simultaneously, connected, and learning from each other.

## Three layers required

**Sensing** — the building has to know what's happening in real time. Resident data, maintenance tickets, lease data, occupancy, financials, event attendance. Most of this already exists. The question is whether agents can read it when they need to.

**Acting** — agents need the authority to act, not just recommend. Sending communications, updating pricing, dispatching vendors, adjusting spend. Human-in-the-loop gets replaced by exception-based oversight: the agent acts, humans review the exceptions.

**Learning** — actions connect to outcomes. What worked gets propagated across properties. What didn't gets discarded. This is the Company Cortex layer — the thing that turns a collection of automated loops into a system that actually gets smarter over time.

## Where Flow is today

Islands of sensing exist — Sage, Ramp, Toast, property P&L. The beginning of acting exists — agents flag anomalies, the intranet surfaces data. The learning layer doesn't exist yet.

The leasing agent work Charles's team is doing is the first real proof of concept. Tasks documented, skills built, headcount eliminated. That's the pattern. The self-managed building gets built one function at a time, starting where tasks are discrete and outcomes are measurable.
